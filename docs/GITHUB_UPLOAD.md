# Загрузка проекта на GitHub

## Вариант 1 — через сайт GitHub

1. Создайте новый репозиторий.
2. Распакуйте архив проекта.
3. Нажмите **Add file → Upload files**.
4. Загрузите содержимое папки так, чтобы `index.html` находился в корне репозитория.
5. Выполните commit.

Не загружайте только ZIP, если планируете GitHub Pages: Pages должен видеть распакованный `index.html`.

## Вариант 2 — через Git в Termux

Установите Git:

```bash
pkg install git -y
```

Перейдите в папку:

```bash
cd ~/storage/downloads/stalker-pda-android-github-release
```

Инициализируйте репозиторий:

```bash
git init
git add .
git commit -m "STALKER PDA Android release"
git branch -M main
git remote add origin https://github.com/USERNAME/REPOSITORY.git
git push -u origin main
```

GitHub может запросить Personal Access Token или SSH-ключ вместо пароля.

## Включение GitHub Pages

1. Откройте репозиторий.
2. Перейдите в **Settings → Pages**.
3. В **Build and deployment** выберите **Deploy from a branch**.
4. Выберите ветку `main` и папку `/ (root)`.
5. Сохраните.
6. Дождитесь публикации.

Адрес будет примерно таким:

```text
https://USERNAME.github.io/REPOSITORY/
```

GitHub Pages использует HTTPS, поэтому Geolocation API может показать запрос разрешения. Пользователь всё равно должен разрешить геоданные в браузере.

## Localhost или GitHub Pages

### Localhost

Плюсы:

- работает локально на смартфоне;
- сервер слушает только loopback;
- не требует публикации приложения;
- подходит для полевой работы после установки пакетов Termux.

Открывать:

```text
http://localhost:8080/
```

### GitHub Pages

Плюсы:

- не нужен Termux после публикации;
- приложение открывается по HTTPS;
- проще обновлять для нескольких устройств.

Минусы:

- первый запуск требует интернет;
- код и документация публичны в открытом репозитории;
- карта всё равно выбирается локально пользователем и не хранится в репозитории, если вы её туда сами не добавили.

## Что не следует публиковать

- реальные карты, которые вы не хотите делать публичными;
- JSON с личными заметками и координатами;
- резервные копии из полевого использования;
- секретные токены GitHub;
- файлы с персональными данными.
