## 1. Почему нужен localhost

Geolocation API относится к возможностям, доступным в защищённом контексте. Современные браузеры обычно считают `http://localhost` и loopback-адреса доверенными для локальной разработки.

Приложение необходимо открывать так:

```text
http://localhost:8080/
```

Неправильные варианты:

```text
content://media/external/...
file:///storage/emulated/0/Download/index.html
```

При прямом открытии HTML через файловый менеджер Chrome может сразу вернуть отказ и не показать системное окно разрешения геоданных.

![Правильный адрес](images/guide-03-open-localhost.png)

## 2. Установка Termux

Termux — терминал и Linux-среда для Android. Root-доступ не требуется.

Используйте один официальный источник:

- F-Droid: `https://f-droid.org/packages/com.termux/`
- GitHub Releases: `https://github.com/termux/termux-app/releases`

Не смешивайте сборки Termux и его дополнений из разных источников: Android не позволит обновлять приложение с другой подписью.

![Установка Termux](images/guide-01-install-termux.png)

После установки откройте Termux и дождитесь появления командной строки.

## 3. Распаковка проекта

### Вариант A — через файловый менеджер

1. Скачайте архив `stalker-pda-android-github-release.zip`.
2. Откройте папку `Download`.
3. Распакуйте архив.
4. Убедитесь, что внутри папки есть `index.html`, `server.py` и `start-android.sh`.

Ожидаемый путь:

```text
Download/stalker-pda-android-github-release/index.html
```

### Вариант B — через Termux

Сначала разрешите Termux доступ к общей памяти:

```bash
termux-setup-storage
```

Android покажет запрос. Разрешите доступ.

Затем:

```bash
cd ~/storage/downloads
unzip stalker-pda-android-github-release.zip
cd stalker-pda-android-github-release
```

Проверьте файлы:

```bash
ls
```

В списке должны быть:

```text
index.html
server.py
start-android.sh
README.md
GUIDE_ANDROID.html
```

## 4. Установка Python

В Termux выполните:

```bash
pkg update -y
pkg install python unzip -y
```

Проверка:

```bash
python --version
```

Должна появиться версия Python без сообщения об ошибке.

## 5. Запуск локального сервера

Перейдите в папку проекта:

```bash
cd ~/storage/downloads/stalker-pda-android-github-release
```

Запустите:

```bash
bash start-android.sh
```

Либо напрямую:

```bash
python server.py --port 8080
```

Ожидаемый вывод:

```text
STALKER PDA запущен локально.
Откройте в Chrome именно: http://localhost:8080/
Не открывайте index.html через file:// или content://.
Для остановки сервера нажмите Ctrl+C.
```

![Команды запуска](images/guide-02-start-server.png)

### Другой порт

Если порт 8080 занят:

```bash
bash start-android.sh 8081
```

Тогда открывайте:

```text
http://localhost:8081/
```

Сервер намеренно привязан к `127.0.0.1`. Это означает, что он доступен только на этом смартфоне, а не другим устройствам в сети.

## 6. Открытие в Chrome

1. Не закрывайте Termux — просто переключитесь в Chrome.
2. Введите в адресную строку:

```text
http://localhost:8080/
```

3. Нажмите Enter.
4. Должен открыться экран включения КПК.
5. Нажмите **Включить КПК**.

Не используйте кнопку «Открыть с помощью Chrome» в файловом менеджере: она обычно создаёт адрес `content://...`.

## 7. Разрешение геоданных

### Перед запуском GPS

1. Откройте шторку Android.
2. Включите **Местоположение / Геолокация**.
3. Проверьте разрешение Chrome:
   - Android → Настройки;
   - Приложения → Chrome;
   - Разрешения → Местоположение;
   - выберите «Разрешать только во время использования»;
   - включите точное местоположение, если пункт доступен.

### Запрос внутри приложения

1. Откройте КПК через `localhost`.
2. Нажмите **GPS**.
3. Chrome должен показать запрос.
4. Выберите разрешение геоданных при использовании приложения.

![Пример разрешения](images/guide-04-gps-permission.png)

### Если запрос уже был отклонён

1. Оставаясь на `http://localhost:8080/`, нажмите значок слева от адресной строки.
2. Откройте **Разрешения** или **Настройки сайта**.
3. Найдите **Местоположение**.
4. Выберите «Разрешить» либо выполните «Сбросить разрешения».
5. Обновите страницу и снова нажмите GPS.

Пункт меню зависит от версии Chrome и оболочки Android.

## 8. Первичная настройка КПК

1. Нажмите **Карта**.
2. Выберите растровое изображение: JPG, PNG, WebP, GIF, BMP или AVIF.
3. Дождитесь отображения карты.
4. Проверьте:
   - карта расположена по центру;
   - кнопки `−` и `+` меняют масштаб;
   - перетаскивание работает;
   - при увеличении нельзя утащить изображение дальше его границ;
   - кнопка **Центр** возвращает карту к вписанному положению.

Для теста в архиве есть:

```text
examples/demo-map.png
```

![Основные кнопки](images/guide-05-app-controls.png)

При загрузке каждой новой карты приложение заново определяет её реальную ширину и высоту, рассчитывает минимальный масштаб и новые ограничения перемещения.

## 9. GPS-привязка A/B

Обычный JPG не содержит сведений о том, где он находится на Земле. Поэтому приложение использует две контрольные точки.

### Точка A

1. Встаньте в месте, которое легко найти на карте: перекрёсток, поворот дороги, здание или другой объект.
2. Включите GPS и дождитесь координат.
3. Нажмите **Привязка** или в разделе «Данные» — **Привязать точку A**.
4. Коснитесь точного места на карте.

### Точка B

1. Переместитесь в другое известное место.
2. Желательно увеличить расстояние между A и B. Слишком близкие точки сильнее реагируют на ошибку GPS.
3. Нажмите **Точка B** / **Привязать точку B**.
4. Коснитесь соответствующего места на карте.

После сохранения B:

- направление и масштаб вычисляются по паре координат;
- GPS-точка начинает перемещаться по карте;
- служебные маркеры A/B через короткое время исчезают;
- сама привязка остаётся сохранённой.

### Кнопка «Я»

Кнопка **Я** центрирует карту на текущей GPS-позиции только один раз. После этого ручное перемещение карты не отменяется новыми GPS-сигналами.

## 10. Метки, задания и заметки

### Новая метка

1. Нажмите **Метка**.
2. Коснитесь нужного места карты.
3. Заполните название и описание.
4. Выберите тип и статус.
5. Нажмите **Сохранить**.

Доступные типы включают задачу, аномалию, тайник, переход, торговца, группировку, опасность, лабораторию и заметку.

### Задания

На вкладке **Задания** доступны фильтры:

- все;
- активные;
- выполненные.

### Заметки

На вкладке **Заметки** можно создать личную запись без привязки к карте.

![Метки и данные](images/guide-06-calibration-backup.png)

## 11. Экспорт и восстановление

Откройте вкладку **Данные**.

### Экспорт

Нажмите **Экспорт**. Будет создан JSON-файл с:

- метками и заданиями;
- заметками;
- GPS-привязкой и ручными параметрами.

Файл карты в JSON не включается. Храните исходное изображение карты отдельно.

### Импорт

1. Нажмите **Импорт**.
2. Выберите ранее сохранённый JSON.
3. Подтвердите замену текущих данных.
4. При необходимости снова выберите файл карты с устройства.

---

## 12. Остановка и повторный запуск

### Остановить сервер

Вернитесь в Termux и нажмите:

```text
Ctrl+C
```

### Запустить снова

```bash
cd ~/storage/downloads/stalker-pda-android-github-release
bash start-android.sh
```

После этого снова откройте:

```text
http://localhost:8080/
```

### Если Android закрывает Termux

В настройках Android для Termux можно выбрать режим батареи «Без ограничений» или «Не оптимизировать». Название зависит от производителя телефона.

---

## 13. Обновление версии

Перед обновлением:

1. Откройте **Данные**.
2. Выполните экспорт JSON.
3. Сохраните исходную карту.
4. Остановите сервер.
5. Замените папку проекта новой версией.
6. Запустите сервер.
7. При необходимости импортируйте JSON и выберите карту.

Не удаляйте старую версию, пока новая не проверена.

---

## 14. Проверка правильной работы

### Сервер

- Termux продолжает работать;
- в нём видны запросы вида `GET /`;
- Chrome открывает `http://localhost:8080/`.

### GPS

- системная геолокация включена;
- Chrome имеет разрешение;
- после нажатия GPS отображаются широта, долгота и точность;
- после A/B точка перемещается по карте.

### Карта

- новая карта центрируется;
- при увеличении нельзя выйти за края изображения;
- при изменении ориентации границы сохраняются;
- **Я** центрирует один раз;
- последующее ручное перетаскивание не вызывает прыжков обратно.

### Данные

- метка сохраняется;
- заметка сохраняется;
- экспорт создаёт JSON;
- импорт восстанавливает данные;
- после перезапуска браузера сохранённые данные доступны, если Chrome не очищал хранилище сайта.
