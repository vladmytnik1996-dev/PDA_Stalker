# Устранение неполадок

## Chrome пишет `ERR_CONNECTION_REFUSED`

Причина: сервер не запущен, Termux был закрыт или указан другой порт.

Проверка:

```bash
cd ~/storage/downloads/stalker-pda-android-github-release
bash start-android.sh
```

После запуска откройте тот адрес, который показан Termux.

---

## Порт 8080 занят

Запустите другой порт:

```bash
bash start-android.sh 8081
```

Откройте:

```text
http://localhost:8081/
```

---

## Команда `cd ~/storage/downloads/...` не работает

1. Выполните:

```bash
termux-setup-storage
```

2. Разрешите Android доступ к файлам.
3. Проверьте:

```bash
ls ~/storage/downloads
```

4. Убедитесь, что название распакованной папки совпадает с командой.

---

## GPS сразу пишет «запрещено пользователем»

Проверьте четыре уровня:

1. Адрес — только `http://localhost:8080/`, а не `content://` или `file://`.
2. Android → системная геолокация включена.
3. Android → Приложения → Chrome → Разрешения → Местоположение разрешено.
4. Chrome → localhost → Настройки сайта → Местоположение разрешено или сброшено.

После изменения разрешений полностью обновите страницу и снова нажмите GPS.

---

## Окно разрешения не появляется

Обычно это означает, что решение уже сохранено браузером.

1. Откройте localhost.
2. Нажмите значок слева от адреса.
3. Откройте разрешения сайта.
4. Сбросьте разрешения.
5. Обновите страницу.
6. Нажмите GPS внутри приложения — запрос появляется только после вызова геолокации.

---

## GPS долго не получает координаты

- выйдите на открытое место;
- подождите 10–60 секунд после первого включения;
- включите точное местоположение Android;
- отключите режим энергосбережения;
- убедитесь, что Chrome находится на экране;
- попробуйте остановить и снова включить GPS.

---

## GPS-точка двигается неточно

- проверьте, что A и B поставлены на действительно узнаваемые места;
- увеличьте расстояние между A и B;
- не используйте две почти совпадающие точки;
- учитывайте, что нарисованная или игровая карта может быть геометрически искажена;
- при сильной ошибке выполните **Сброс GPS** и калибровку заново.

---

## Карта не сохраняется после закрытия Chrome

Возможные причины:

- браузер запретил IndexedDB;
- включён приватный режим;
- недостаточно места;
- Android очистил данные Chrome;
- браузер автоматически удаляет данные сайта.

Задания и заметки регулярно экспортируйте в JSON. Исходный файл карты храните отдельно.

---

## Карта слишком большая или не открывается

Приложение ограничивает небезопасные и чрезмерно тяжёлые изображения. Используйте JPG/PNG/WebP разумного разрешения. При необходимости уменьшите изображение в графическом редакторе.

---

## Карта после загрузки выглядит слишком маленькой

Это нормальное начальное вписывание. Используйте `+` или жест двумя пальцами. Перемещение останется ограничено краями изображения.

---

## Карта не перемещается по одной из осей

Если изображение после масштабирования всё ещё меньше viewport по этой оси, приложение удерживает его по центру. Увеличьте масштаб — после этого появится допустимый диапазон перемещения.

---

## Android закрывает сервер в фоне

Откройте:

```text
Android → Настройки → Приложения → Termux → Батарея
```

Выберите «Без ограничений» / «Не оптимизировать», если такой режим доступен. Производители Android называют этот пункт по-разному.

---

## После обновления исчезли данные

Импортируйте JSON-резервную копию и снова выберите карту. Не очищайте данные сайта `localhost` до экспорта.

---

## Нужно проверить сервер вручную

В Termux откройте второй сеанс и выполните:

```bash
python -c "import urllib.request; print(urllib.request.urlopen('http://localhost:8080/').status)"
```

Ожидается:

```text
200
```
